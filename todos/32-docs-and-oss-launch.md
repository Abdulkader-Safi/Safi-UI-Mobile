# 32 — Docs site, contribution guide, and v1.0 release

**Phase:** 7 — OSS Launch
**PRD refs:** §16 (Phase 7), §17

## Goal

Ship the public-facing artifacts that turn the library into an adoptable open-source project.

## Deliverables

- Docs site (`docs/`, rspress) updated to remove every WIP marker now that code has landed
- Component reference page autogenerated from prop tables in each component's Rust file
- `CONTRIBUTING.md` covering: dev setup, running tests, the §6.10 hot-reload flow, the §14.1 panic-handler hook
- `CODE_OF_CONDUCT.md`
- `CHANGELOG.md` seeded with v1.0
- MIT `LICENSE` (already in `00`; verify it ships)
- GitHub release `v1.0.0` with cross-compiled binaries for the example apps
- `cargo publish` for `safi-ui` and `safi-ui-macros` to crates.io
- Update `docs/docs/status.md` to "shipped"
- Launch blog post / Show HN draft

## Dependencies

- `31` (and transitively everything earlier)

## Acceptance

- All v1 success-metric targets verified against the shipped binaries (§17.1):
  - Cold parse < 5ms / p99
  - Layout < 2ms / 50 nodes
  - Frame GPU < 4ms
  - Static-frame CPU < 0.1ms
  - Binary size < 800KB stripped (arm64 worst case)
  - ≥ 30 built-in components
- Crates published; docs site live; GitHub release tagged
